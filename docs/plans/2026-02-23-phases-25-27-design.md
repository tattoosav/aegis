## Phase 25: Production Deployment

### 25.1 PyInstaller Build System

**Files:** `build/build.py`, `build/aegis.spec`

A build script that freezes `src/aegis/` into a standalone directory:

- Uses PyInstaller to bundle Python 3.11+ runtime and all dependencies (ZeroMQ, PySide6,
  numpy, scikit-learn, ONNX Runtime, cryptography, pefile, pywintrace)
- Bundles non-Python assets: YARA rules (`rules/`), LOLBins YAML, Sysmon config, icons
- Produces `dist/aegis/` with `aegis.exe` entry point
- `.spec` file for reproducible builds
- Hidden import hooks for PySide6 plugins, ctypes DLLs, ETW providers
- UPX compression for smaller installer size
- Version info embedded from `pyproject.toml` (product name, version, company)

### 25.2 NSIS Installer

**Files:** `build/installer.nsi`

Standard Windows installer wizard:

- **Welcome page** — Aegis branding and description
- **License page** — Project license agreement
- **Install directory** — Default `C:\Program Files\Aegis`
- **Component selection** — Core (required), Sysmon (default checked), Desktop Shortcut
- **Installation actions:**
  - Copy `dist/aegis/` to install directory
  - Install Sysmon from bundled `tools/sysmon/` with SwiftOnSecurity config
  - Register Aegis as Windows Service (`sc create AegisDefense`)
  - Set service to Automatic (delayed) start
  - Create Start Menu and optional Desktop shortcuts
  - Write uninstall registry keys
- **Finish page** — "Launch Aegis" checkbox
- **Uninstaller** — Stop/remove service, delete files, remove registry, optionally remove Sysmon

### 25.3 Windows Service Integration

**Files:** Modify `src/aegis/core/service.py`

Full Windows Service using pywin32 `ServiceFramework`:

- **Service name:** `AegisDefense`
- **Display name:** `Aegis Security Defense System`
- **Startup type:** Automatic (delayed start — waits for network)
- **Recovery:** Restart on 1st failure, restart on 2nd, do nothing on 3rd
- `SvcDoRun()` starts `AegisCoordinator` (all sensors, detection, response)
- `SvcStop()` gracefully shuts down coordinator
- Windows Event Log integration for service start/stop/errors
- **Dual-mode:** Detects at startup whether running as service or desktop app (tray mode)

### 25.4 First-Run Wizard

**Files:** Create `src/aegis/ui/first_run_wizard.py`

PySide6 wizard dialog on first startup (8 pages):

1. **Welcome** — Aegis branding, "Let's set up your protection"
2. **Sensor Configuration** — Toggle sensors on/off, adjust intervals (all on by default)
3. **Threat Intelligence** — Toggle feeds, optional VirusTotal API key, optional Claude API key
4. **Detection Tuning** — Sensitivity slider (Low/Medium/High) mapping to anomaly thresholds
5. **Sysmon Status** — Check/install Sysmon, show config preview
6. **Baseline Scan** — Quick system inventory: running processes, network connections, installed
   software. Establishes initial "normal" for the machine.
7. **Exclusions** — Add trusted directories, processes, or IPs to exclude from monitoring
8. **Summary & Launch** — Show configuration summary, "Start Protection" button

Writes config to `aegis.yaml`. Sets `first_run_complete: true` flag.

### 25.5 Sysmon Bundling

**Files:** `tools/sysmon/`, modify `build/build.py`

- Bundle Sysmon 15.x binary in `tools/sysmon/Sysmon64.exe`
- Bundle SwiftOnSecurity config in `tools/sysmon/sysmonconfig.xml`
- Install command: `Sysmon64.exe -accepteula -i sysmonconfig.xml`
- Installer and first-run wizard both offer Sysmon installation
- Verify Sysmon status via `sc query Sysmon64` before attempting install

## Phase 26: AI/ML Enhancement

### 26.1 Adaptive Baselines

**Files:** Create `src/aegis/ml/training_pipeline.py`, `src/aegis/ml/feature_extractor.py`

Per-machine learning of "normal" behavior:

- **Baseline collection:** First 24-72 hours (configurable), collect per-sensor metrics:
  process creation rates, network patterns, file change frequencies, DNS queries
- **Isolation Forest retraining:** After baseline period, retrain on local data using
  scikit-learn. Store model in `data/models/` using joblib. Retrain weekly on rolling
  7-day window.
- **Autoencoder fine-tuning:** Transfer learning — freeze early layers, retrain final layers
  on local event feature vectors. Export to ONNX for inference.
- **Feature extraction:** Standardized pipeline converting raw events to numeric vectors
  (connection counts, entropy, timing patterns, process tree depth, registry change rate)
- **Model versioning:** Each model gets version + timestamp. Auto-rollback if new model
  produces >2x alert volume vs. previous.
- **Dashboard integration:** Training status, model age, accuracy metrics on Settings page

### 26.2 Concept Drift Detection

**Files:** Create `src/aegis/ml/drift_detector.py`

Monitor for significant environment changes that invalidate the learned baseline:

- Track feature distribution statistics (mean, std, quantiles) per metric over time
- Use Page-Hinkley or ADWIN algorithm to detect distribution shifts
- When drift detected: generate advisory alert, offer to retrain models
- Examples: new software installed, network topology changed, user behavior shift
- Configurable sensitivity for drift detection

### 26.3 Threat Prediction Engine

**Files:** Create `src/aegis/detection/threat_predictor.py`

Sequence-based attack step prediction:

- **Attack sequence database:** Known MITRE ATT&CK chains as directed graphs
  (Initial Access -> Execution -> Persistence -> Privilege Escalation -> ...)
- **LSTM sequence model:** Trained on technique ID sequences. Given observed techniques,
  predicts next likely technique with probability.
- **Confidence calibration:** Platt scaling ensures prediction probabilities are
  well-calibrated (predicted 80% = actual 80% occurrence rate)
- **Predictive alerts:** When confidence > 0.7, generate "Predicted Threat" alert:
  "Based on T1566 -> T1059, 78% likelihood of T1055 next"
- **Defensive recommendations:** Map predicted techniques to defensive actions
- **Pipeline integration:** Parallel engine consuming correlated incident data

### 26.4 LLM-Powered Analysis

**Files:** Create `src/aegis/intelligence/llm_analyzer.py`

Deep Claude API integration:

- **Automated triage:** On alert, send context (event data, related events, process tree,
  connections) to Claude. Returns: severity assessment, attack narrative, investigation
  steps, false-positive likelihood.
- **Natural language hunting:** Users type questions -> Claude converts to SQL -> execute
  against SQLite -> present results with analysis. Show generated SQL for transparency.
- **Incident summarization:** On incident close, generate plain-English report with
  timeline, impact assessment, remediation summary.
- **Offline fallback:** Falls back to template-based explainer when API unavailable.
  All LLM features are optional and degradable.
- **Privacy controls:** Config for what data can be sent to API. Anonymize sensitive
  fields (paths, usernames). Option for local LLM via Ollama.
- **Rate limiting:** Configurable budget (default 100 calls/day). Batch low-priority
  triages. High-severity alerts get immediate calls.

## Phase 27: Threat Hunting & Forensics

### 27.1 Attack Timeline Reconstruction

**Files:** Create `src/aegis/forensics/timeline_engine.py`,
  `src/aegis/ui/widgets/attack_timeline.py`

Visual attack timeline:

- **Data model:** `TimelineEvent` with timestamp, source sensor, event type, MITRE
  technique, process context, network context, causality links (parent event ID)
- **Reconstruction:** Given incident ID, pull correlated events, sort chronologically,
  compute causality arrows (process A -> process B -> network C)
- **UI widget:** Horizontal timeline with:
  - Color-coded dots (green=info, yellow=suspicious, red=malicious)
  - MITRE technique labels above events
  - Causality arrows connecting related events
  - Click-to-expand event details
  - Zoom in/out on time ranges
- **Export:** Timeline as PNG image or interactive HTML

### 27.2 Report Generation

**Files:** Enhance `src/aegis/response/report_generator.py`,
  create `templates/reports/`

Multi-format security reports:

- **Report types:**
  - Incident Report: timeline, assets, IOCs, MITRE techniques, responses, remediation
  - Daily Summary: alert counts, top rules, new IOCs, sensor health
  - Threat Assessment: security posture score, trending threats, exposure
- **Output formats:** HTML (styled, printable), PDF (WeasyPrint), JSON, CSV
- **Template system:** Jinja2 templates in `templates/reports/`, customizable branding
- **Automated scheduling:** Daily summary at midnight via `TaskScheduler`.
  Incident reports on incident close.
- **Dashboard integration:** "Generate Report" button on Alerts and Threat Hunt pages

### 27.3 Natural Language Threat Hunting UI

**Files:** Complete `src/aegis/ui/pages/threat_hunt.py`

Full threat hunting interface:

- **NL query input:** Text field for plain English questions:
  "What processes connected to Russia in the last 24 hours?"
- **Query translation:** LLM converts NL to SQL against Aegis schema.
  Shows generated SQL for transparency.
- **Results display:** Sortable table with expandable rows, links to related alerts
- **Saved queries:** Bookmark frequently-used queries
- **Query history:** Searchable history of past queries and results
- **Pre-built library:** Common hunting queries (lateral movement, data exfil,
  persistence, credential access)
- **Offline mode:** Falls back to direct SQL mode when no LLM available

## Success Criteria

### Phase 25
- `aegis-setup.exe` installs Aegis, Sysmon, and registers the service
- Aegis starts on boot as a Windows Service
- First-run wizard configures sensors, feeds, and detection sensitivity
- Uninstaller cleanly removes everything

### Phase 26
- Isolation Forest retrains on local data after baseline period
- Alert false-positive rate drops >50% after baseline learning
- Threat predictor achieves >60% accuracy on known attack sequences
- Claude API triage provides actionable investigation steps
- All ML features degrade gracefully when unavailable

### Phase 27
- Attack timeline renders incident events chronologically with causality
- Reports export as HTML and PDF with professional formatting
- NL hunting converts plain English to correct SQL >80% of the time
- Pre-built query library covers top 20 threat hunting scenarios
